## 🔄 工作流程

```
┌─────────────────────────────────────────────┐
│ 1. 需求分析 (@pm)                            │
│    ↓                                         │
│ 2. 架构设计 (@architect)                      │
│    ↓                                         │
│ 3. 任务规划 (task_plan skill)                 │
│    ↓                                         │
│ 4. 并行开发                                  │
│    ├─ @backend-dev (API)                     │
│    └─ @frontend-dev (UI)                     │
│    ↓                                         │
│ 5. 并行测试                                  │
│    ├─ @whitebox-qa (单元/接口测试)           │
│    └─ @blackbox-qa (E2E测试)                 │
│    ↓                                         │
│ 6. Bug 修复循环                              │
│    ├─ 发现 Bug → 通知开发                     │
│    ├─ 开发修复 → 重新测试                     │
│    └─ 重复直到全部通过                        │
│    ↓                                         │
│ 7. 代码审查 (@pr-reviewer)                   │
│    ↓                                         │
│ 8. 文档更新 (@pm + @architect)               │
│    ↓                                         │
│ 9. 代码提交 (@committer)                     │
│    ↓                                         │
│ 10. 发布验收                                  │
└─────────────────────────────────────────────┘
```
