# Sisyphus-X 全量对齐需求文档 — 设计方案

> 日期: 2026-03-05
> 状态: 已批准
> 背景: 当前实现与需求文档(docs/Sisyphus-X需求文档.md)存在大量差距，需全量对齐

---

## 一、整体策略

采用**自底向上、按模块分批次**实施：先修全局基础设施，再按依赖关系逐模块对齐。
顺序：全局基础 → 项目管理 → 关键字配置 → 接口定义 → 场景编排 → 测试计划 → 测试报告

---

## 二、Phase 1: 全局基础设施

### 2.1 分页组件增强

**当前问题**: `Pagination.tsx` 固定 `size=10`，不支持每页条数切换。

**改动**:
- 增加 `onSizeChange` 回调、`pageSizeOptions` prop（默认 `[10, 20, 50, 100]`）
- 左侧显示「共 X 条」，右侧增加每页条数下拉
- 切换条数时自动重置 page 为 1
- 所有使用分页的页面同步接入：
  - `pages/api-automation/ProjectManagement.tsx`
  - `pages/keywords/KeywordManagement.tsx`
  - `pages/scenario/index.tsx`
  - `pages/plans/TestPlan.tsx`
  - `pages/reports/TestReport.tsx`
  - `pages/api-automation/DatabaseConfigList.tsx`

### 2.2 搜索框回车支持

所有列表页的搜索框确认支持回车触发搜索。

### 2.3 页面风格一致性检查

确认所有列表页使用统一的 header/spacing/typography 模式。

## 五、Phase 4: 接口定义重构

**需求文档**: 6.3

### 5.1 布局重构

```
┌──────────────────────────────────────────────────────────────────┐
│  页头: 接口定义 │ 项目: [下拉▼] │ [环境: dev ▼] [环境管理]        │
├──────────┬───────────────────────────────────────────────────────┤
│ 接口目录树 │ 请求编辑器（上半部分）                                 │
│ (w-64)   │ [Method▼] [URL] [发送]                               │
│          │ [Params] [Auth] [Headers] [Body]                     │
│          ├───────────────────────────────────────────────────────┤
│ [+目录]  │ 响应查看器（下半部分）                                 │
│ [+接口]  │ [Body] [Headers] [Cookies] [Timeline] [Logs]         │
│ [↑cURL]  │                                                      │
│ [↑Swagger]│                                                     │
└──────────┴───────────────────────────────────────────────────────┘
```

关键点:
- 页头统一风格（同 padding/typography）
- 项目选择器在页头左侧
- 环境选择器在页头右侧
- 主区域改为**上下分割**（请求在上、响应在下），支持拖拽调整分割比例

### 5.2 功能修复

- **环境变量替换**: `handleSend` 中实现 `{{变量}}` 替换和域名拼接
- **多级目录**: `buildTree` 支持无限层级嵌套
- **拖拽排序**: 对接后端排序 API（目录间、接口间排序）
- **环境管理弹窗**: 增加「全局变量」区域 + 变量描述字段
- **右键菜单**: 重命名、删除文件夹、复制接口
- **数据持久化**: Auth 配置、FormData 保存到接口数据

### 5.3 Bug 修复

- `keyValuePairsToObject` 使用 `trimmedKey`
- InterfaceTree 按钮路径一致性
- TypeScript 类型安全

## 六、Phase 5: 场景编排重写

**需求文档**: 6.4
**这是最大的改动**: 从 ReactFlow 画布编辑器重写为列表式步骤编辑器。

### 6.1 场景列表页

- 接入增强分页组件
- 搜索回车支持
- 无其他大改动

### 6.2 场景编辑页 — 完全重写

```
┌───────────────────────────────────────┬─────────────────┐
│ 页头: [←] 场景编排 - {名称}           │ [保存][调试][关闭]│
├───────────────────────────────────────┼─────────────────┤
│ 基础信息 (可折叠)                      │ 右侧边栏 (w-72) │
│ 项目[▼] 名称[___] 描述 优先级 标签     │ 运行环境        │
│                                       │ [环境▼] [管理]   │
│ 场景信息 Tab:                          │                 │
│ [测试步骤] [变量] [前置SQL] [后置SQL]   │ 测试数据集      │
│                                       │ [数据集▼] [新建] │
│ [+添加步骤]                            │                 │
│                                       │                 │
│ 步骤1 ☰ (拖拽)                        │                 │
│ ┌───────────────────────────────────┐ │                 │
│ │ 描述: [____]                      │ │                 │
│ │ 关键字: [类型▼] → [名称▼] → [...] │ │                 │
│ │ 参数区 (根据关键字类型动态渲染)     │ │                 │
│ │ 断言 [+添加]                      │ │                 │
│ │ 变量提取 [+添加]                   │ │                 │
│ └───────────────────────────────────┘ │                 │
│                                       │                 │
│ 步骤2 ☰                              │                 │
│ ┌───────────────────────────────────┐ │                 │
│ │ ...                               │ │                 │
│ └───────────────────────────────────┘ │                 │
└───────────────────────────────────────┴─────────────────┘
```

#### 步骤卡片

- 可折叠：折叠态只显示描述+关键字名+状态；展开态显示完整配置
- 左侧拖拽手柄（`@dnd-kit/sortable`）
- 右上角删除按钮

#### 级联关键字下拉

- L1: 关键字类型（发送请求/断言/提取变量/数据库/自定义）
- L2: 关键字名称（按类型过滤，从 keywordsApi 动态加载启用的关键字）
- L3: (仅发送请求→HTTP请求) 接口选择（当前项目接口，支持目录→接口名级联筛选+搜索）

#### 参数区动态渲染

- 发送请求→HTTP请求: 自动回填 Method/URL/Headers/Body，用户可修改
- 断言→JSON: JSONPath + 判断符号下拉 + 预期值
- 提取变量→JSON: 变量名 + 类型(全局/环境) + JSONPath
- 数据库操作: 数据源下拉 + SQL 编辑器（Monaco）
- 自定义关键字: 按定义的入参动态生成表单

#### 右侧边栏

- 运行环境下拉 + 环境管理弹窗按钮
- 测试数据集下拉 + 新建数据集抽屉

#### Tab 区

- 测试步骤（步骤卡片列表）
- 变量定义（场景级 KV 编辑器）
- 前置 SQL（Monaco 编辑器，支持 `{{变量}}`）
- 后置 SQL（Monaco 编辑器，支持 `{{变量}}`）

#### 删除/复用

- 删除: `nodes/` 目录、ReactFlow 画布、flowToSteps/stepsToFlow、reactflow 依赖
- 复用: 场景列表页、API 客户端、DatasetSidebar/Dialog/TableEditor、保存/调试逻辑

### 7.2 实时执行报告页（6.5.5 — 重大改造）

```
┌──────────────────────────────────────────────────────────────────┐
│ 面包屑: 测试计划 / {计划名称}           │ [暂停] [终止]           │
├──────────────────────────────────────────────────────────────────┤
│ 统计区: [运行动画] | 场景:总/通过/失败 | 接口:总/通过/失败 | 性能  │
├──────────────────────────────────────────────────────────────────┤
│ 场景列表 (三层折叠)                                               │
│ ▶ ✅ 场景1 [████████████] 100%                                  │
│ ▼ ❌ 场景2 [████████░░░░]  60%                                  │
│   ├─ ✅ POST /login       200 320ms                              │
│   ├─ ❌ POST /order/create 500 1.2s                              │
│   │   ┌ Request/Response 详情 ┐                                  │
│   └─ ⏳ DELETE /cart (等待中)                                     │
│ ⏳ 场景3 [░░░░░░░░░░░░]  0%                                     │
├──────────────────────────────────────────────────────────────────┤
│ [再次运行] [导出报告] [Allure报告]                                │
└──────────────────────────────────────────────────────────────────┘
```

#### 三层折叠

- L1: 场景（状态+名称+进度条+通过率）— 默认折叠
- L2: 接口请求（方法+URL+状态码+耗时）— 场景展开后显示
- L3: 请求详情（Request/Response）— 点击接口展开

#### WebSocket 增量渲染

- WS 消息直接包含步骤级数据，前端增量更新本地状态
- 后端推送接口级进度（每个请求的开始/完成/失败）
- 不再依赖 refetchExecution 轮询

#### 后端改造

- `plans.py` 改用 `ExecutorAdapter` 替代 `EngineExecutor`
- 修复 session 生命周期（后台任务独立创建 session）
- 推送接口级粒度消息

#### 操作按钮

- 暂停: `POST /plans/{id}/pause`（后端已有）
- 恢复: 暂停后显示，调用 `POST /plans/{id}/resume`
- 再次运行: 调用 `POST /plans/{id}/execute` 并导航到新执行页
- 导出/Allure: 对接后端 API

## 九、技术决策摘要

| 决策 | 选择 | 原因 |
|------|------|------|
| 实施策略 | 自底向上按模块 | 被依赖模块先完成，基础设施只改一次 |
| 场景编排 | 重写为列表式步骤编辑器 | 需求文档明确要求，与 YAML 引擎线性执行模型一致 |
| 接口定义布局 | 上下分割（请求上、响应下） | URL/JSON 需要水平宽度，符合 Postman/Insomnia 主流方案 |
| 执行页 WebSocket | 增量渲染 | refetch 轮询效率低，无法实现真正的流式输出 |
| /environments 页面 | 保留 | 用户要求保留系统设置下的独立环境管理入口 |
| 画布编辑器 | 删除 ReactFlow | 用户确认完全重写为列表式 |
