# 计划级聚合报告与完整性验收 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将测试报告严格收敛为“测试计划执行的聚合结果”，并完成深色模式、简体中文下的前端完整性验收与最终视频交付。

**Architecture:** 后端以 `plans.py` 为唯一正式落报告入口，在一次计划执行开始时创建单条 `TestReport`，将所有场景的接口级执行明细统一写入 `TestReportDetail`；前端报告页改为展示计划执行历史与聚合详情，接口页与场景页继续保留调试能力但不落报告。最终通过 `agent-browser` 串起项目、接口、场景、计划、报告的完整验收链路并录制视频。

**Tech Stack:** FastAPI、SQLAlchemy、Pydantic、React、TypeScript、TanStack Query、Playwright/agent-browser、Pytest、Vitest。
