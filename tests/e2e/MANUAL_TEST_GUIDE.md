# Sisyphus-X 主干流程手动测试指南

## 测试前提
- 确保后端运行在 `http://localhost:8000`
- 确保前端运行在 `http://localhost:5173`
- 使用 Chrome/Edge 浏览器
- 打开浏览器开发者工具 (F12) 以查看网络请求

## 测试步骤

### 步骤 A: 新建项目

1. **打开应用**: 访问 `http://localhost:5173`
2. **导航**: 点击左侧菜单 "Project Management" (项目管理)
3. **创建项目**: 
   - 点击右上角 "新建项目" 或 "Create Project" 按钮
   - 填写项目名称: `HP_Manual_Test_[当前时间戳]`
   - 填写项目描述: `手动测试项目`
   - 点击 "创建" 按钮
4. **验证成功**: 
   - 看到成功提示消息
   - 项目列表中出现新创建的项目
   - **截图保存**: `step_a_project_created.png`

**成功证据**: ✅ 项目列表中显示新项目名称

---

### 步骤 B: 新建自定义关键字

1. **导航**: 点击左侧菜单 "Keyword Management" (关键字管理)
2. **创建关键字**:
   - 点击 "创建关键字" 或 "Create Keyword" 按钮
   - 填写关键字名称: `TestKeyword_[当前时间戳]`
   - 填写关键字描述: `手动测试关键字`
   - 代码编辑器: 保持默认代码或输入简单的 Python 代码
   - 点击 "保存" 按钮
3. **验证成功**:
   - 看到成功提示消息
   - 关键字列表中出现新创建的关键字
   - **截图保存**: `step_b_keyword_created.png`

**成功证据**: ✅ 关键字列表中显示新关键字名称

---

### 步骤 C: 创建接口并调试

1. **导航**: 点击左侧菜单 "API Management" (接口管理)
2. **选择项目**: 在页面顶部选择步骤 A 创建的项目
3. **创建接口**:
   - 点击 "新建接口" 或 "New Interface" 按钮
   - 填写接口名称: `TestAPI_[当前时间戳]`
   - 选择请求方法: `GET`
   - 填写 URL: `https://httpbin.org/get`
   - 点击 "保存" 按钮
4. **调试接口**:
   - 点击 "发送" 或 "Send" 按钮
   - 等待响应
5. **验证成功**:
   - 响应区域显示状态码 `200`
   - 响应体显示 JSON 数据
   - **截图保存**: `step_c_api_response.png`

**成功证据**: ✅ 响应区域显示状态码 200 和响应数据

---

### 步骤 D: 新建场景并添加步骤

1. **导航**: 点击左侧菜单 "Scenario Orchestration" (场景编排)
2. **创建场景**:
   - 点击 "新建场景" 或 "New Scenario" 按钮
   - 填写场景名称: `TestScenario_[当前时间戳]`
   - 填写场景描述: `手动测试场景`
   - 选择项目: 选择步骤 A 创建的项目
3. **添加步骤**:
   - 点击 "添加步骤" 或 "Add Step" 按钮
   - 选择步骤 C 创建的接口
   - 确认添加
4. **保存场景**:
   - 点击 "保存" 按钮
5. **验证成功**:
   - 看到成功提示消息
   - 场景列表中出现新创建的场景
   - 场景详情中显示已添加的步骤
   - **截图保存**: `step_d_scenario_created.png`

**成功证据**: ✅ 场景列表中显示新场景,且包含至少一个步骤

---

### 步骤 E: 新建测试计划并执行

1. **导航**: 点击左侧菜单 "Scheduled Tasks" (测试计划)
2. **创建计划**:
   - 点击 "新建计划" 或 "New Plan" 按钮
   - 填写计划名称: `TestPlan_[当前时间戳]`
   - 填写计划描述: `手动测试计划`
   - 选择项目: 选择步骤 A 创建的项目
3. **添加场景**:
   - 点击 "添加场景" 或 "Add Scenario" 按钮
   - 选择步骤 D 创建的场景
   - 确认添加
4. **保存计划**:
   - 点击 "保存" 按钮
5. **执行计划**:
   - 点击 "执行" 或 "Run" 按钮
   - 等待执行完成
6. **验证成功**:
   - 执行状态从 "运行中" 变为 "成功" 或 "完成"
   - 显示执行结果统计
   - **截图保存**: `step_e_plan_executed.png`

**成功证据**: ✅ 计划执行完成,显示最终状态 (成功/失败)

---

### 步骤 F: 查看测试报告详情

1. **导航**: 点击左侧菜单 "Test Reports" (测试报告)
2. **打开报告**:
   - 在报告列表中找到最新的报告 (步骤 E 的执行记录)
   - 点击 "查看" 或 "View" 按钮
3. **验证成功**:
   - 报告详情页显示执行统计 (总数、成功、失败)
   - 显示步骤记录列表
   - 显示每个步骤的执行结果
   - **截图保存**: `step_f_report_detail.png`

**成功证据**: ✅ 报告详情页显示统计数据和步骤记录

## 常见问题排查

### 问题 1: 找不到按钮
- **检查**: 页面是否完全加载
- **检查**: 是否选择了正确的项目
- **检查**: 是否有权限操作

### 问题 2: 表单提交失败
- **检查**: 必填字段是否都已填写
- **检查**: 字段格式是否正确
- **检查**: 开发者工具 Network 标签查看请求详情

### 问题 3: 接口调试无响应
- **检查**: URL 是否正确
- **检查**: 网络连接是否正常
- **检查**: 是否有 CORS 错误

### 问题 4: 计划执行卡住
- **检查**: 后端服务是否正常运行
- **检查**: 是否有 WebSocket 连接错误
- **检查**: 刷新页面查看最新状态

## 附录: API 端点验证

如果需要验证后端 API 是否正常,可以使用以下 curl 命令:

```bash
# 1. 创建项目
curl -X POST http://localhost:8000/api/v1/projects \
  -H "Content-Type: application/json" \
  -d '{"name":"API_Test_Project","description":"API 测试项目"}'

# 2. 获取项目列表
curl http://localhost:8000/api/v1/projects

# 3. 健康检查
curl http://localhost:8000/health
```
