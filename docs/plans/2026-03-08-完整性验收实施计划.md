# 完整性验收 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 完成 Sisyphus-X 前端全量页面完整性验收、主业务闭环深测、必要缺陷修复与最终视频录制交付。

**Architecture:** 先盘点路由、服务与后端接口，再用最小脚本初始化干净验收数据；随后使用 `agent-browser` 进行首轮全量巡检，发现问题后在前后端对应模块做最小修复，最后再次完整回归并录制最终中文深色模式演示视频。整个过程以“可复现、可闭环、可交付”为核心，避免临场随机操作导致结果不稳定。

**Tech Stack:** React 18 + TypeScript + Vite、FastAPI + Python 3.12、agent-browser、项目本地 HTTP API、必要时使用 Pytest/Vitest/项目脚本验证。

---

### Task 1: 盘点运行环境与主路由

**Files:**
- Modify: `docs/plans/2026-03-08-完整性验收设计.md`
- Inspect: `README.md`
- Inspect: `frontend/src/App.tsx`
- Inspect: `frontend/package.json`
- Inspect: `backend/app/main.py`

**Step 1: 确认服务启动方式**

Run: `./sisyphus_init.sh status`
Expected: 能看到前端、后端及依赖服务状态，若未启动则可据此决定启动命令。

**Step 2: 确认前端主路由与模块**

Run: `rg -n "path=|Route path|/keywords|/plans|/reports|/environments|/global-params" frontend/src/App.tsx`
Expected: 列出完整主路由，形成页面验收清单。

**Step 3: 记录需要重点覆盖的页面**

Result: 在执行记录中形成页面清单，包含关键字配置模块与闭环链路页面。

### Task 2: 启动本地服务并验证可访问性

**Files:**
- Inspect: `sisyphus_init.sh`
- Inspect: `backend/.env`
- Inspect: `frontend/.env`

**Step 1: 启动项目服务**

Expected: 返回 200/304/307 等可接受状态，证明服务可访问。

### Task 3: 盘点后端可用验收接口

**Files:**
- Inspect: `backend/app/api/v1/endpoints/`
- Inspect: `backend/app/schemas/`
- Inspect: `backend/app/services/`

**Step 1: 枚举项目、环境、关键字、接口、场景、计划、报告相关端点**

Run: `find backend/app/api/v1/endpoints -type f | sort`
Expected: 找到初始化验收数据所需的 API 文件。

**Step 2: 抽取主要路由前缀与请求模型**

Run: `rg -n "APIRouter|@router\.(get|post|put|patch|delete)" backend/app/api/v1/endpoints backend/app/schemas`
Expected: 得到可用于构造初始化脚本的接口与请求体信息。

### Task 4: 编写验收数据初始化脚本

**Files:**
- Create: `scripts/init_acceptance_data.sh`
- Optionally Create: `tmp/acceptance-data/*.json`
- Inspect: `backend/app/api/v1/endpoints/*`

**Step 1: 先写最小脚本骨架**

```bash
#!/usr/bin/env bash
set -euo pipefail
BASE_URL="${BASE_URL:-http://localhost:8000/api/v1}"
PREFIX="验收项目-20260308"
```

**Step 2: 逐步增加创建项目/环境/关键字/接口/场景/计划的请求**

Expected: 每一步输出新建资源 ID，失败时明确打印响应体。

**Step 3: 运行脚本验证初始化成功**

Run: `bash scripts/init_acceptance_data.sh`
Expected: 输出项目、环境、关键字、接口、场景、计划等资源标识。

### Task 5: 使用 agent-browser 做首轮全量巡检

**Files:**

Run: `agent-browser --session acceptance-20260308 --color-scheme dark open http://localhost:5173`

**Step 2: 对每个主页面执行 snapshot 与核心交互**

**Step 3: 记录所有阻塞/严重/一般问题**
