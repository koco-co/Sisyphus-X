## 1. 项目概述

### 1.1 产品定位

Sisyphus-X 是一款面向测试团队的**自动化测试管理平台**，核心目标是提供从「接口定义 → 场景编排 → 测试计划 → 测试报告」的全链路接口自动化能力，后续将扩展 WEB 自动化和 APP 自动化模块。

### 1.2 核心能力

| 能力域 | 说明 |
|--------|------|
| 接口自动化 | 项目管理、关键字配置、接口定义与调试、场景编排、测试计划执行、测试报告查看 |
| 核心执行器 | 基于 YAML 驱动的 sisyphus-api-engine，支持 CLI 调用、多种报告格式输出 |
| 系统设置 | 全局参数管理（工具函数注册与引用） |
| 扩展预留 | WEB 自动化、APP 自动化、消息通知管理（后续迭代） |

### 1.3 用户角色

| 角色 | 描述 |
|------|------|
| 测试工程师 | 主要使用者，创建项目、定义接口、编排场景、执行测试计划、查看报告 |
| 管理员 | 管理全局参数、系统配置 |

## 2. 技术架构

### 2.1 技术栈总览

| 层级 | 技术栈 |
|------|--------|
| **前端** | Vite + React + TailwindCSS + shadcn/ui + TypeScript |
| **后端** | uv + Python 3.12 + Ruff + Pyright + FastAPI + SQLAlchemy + sisyphus-api-engine |
| **中间件** | PostgreSQL + MinIO + Redis（通过 Docker Compose 管理） |
| **白盒测试** | pytest + requests + uv + Python 3.12 |
| **黑盒测试** | Playwright（+ 待定） |

### 2.2 部署方式

- 中间件（PostgreSQL / MinIO / Redis）通过 `docker-compose.yml` 统一管理。
- 前后端本地开发模式下独立启动。

## 3. 通用规范与约定

> 以下规范适用于全平台所有模块，开发 & 测试均需遵循。

### 3.1 开发规范

| 项目 | 规范 |
|------|------|
| 编码风格 | 全程使用 Google 开发范式 |
| 注释语言 | 中文 |
| 项目结构 | 前端、后端、测试、核心执行器的配置和文档均在同一根目录下维护 |
| 变更记录 | README 和 CHANGELOG 使用同一份文档 |

### 3.2 UI / 交互规范

| 规范项 | 详细说明 |
|--------|----------|
| Toast 提示 | 全局统一风格；位于右上角；需包含**进度条倒计时**效果；提示文案简洁，如「编辑成功」「删除成功」「添加成功」 |
| 分页组件 | 全局统一样式；所有列表页必须支持分页 |
| 查询框 | 全局统一样式, 支持回车和按钮两种模糊查询 |
| 下拉框 | 全局统一样式 |
| 启用/禁用字段 | 全局统一样式（开关按钮） |
| 暂无数据 | 全局统一空状态样式 |
| 删除操作 | 所有删除操作需**二次确认弹窗** |
| 枚举字段筛选 | 所有列表中有枚举值的字段，均需支持**筛选按钮**（含「重置」「确定」操作） |
| 文案规范 | 产品文案需无歧义，全局格式统一 |

### 3.3 标签页机制

**适用范围**: 接口自动化所有子模块。

| 功能 | 说明 |
|------|------|
| 标签页切换 | 支持多标签页打开，点击切换 |
| 关闭当前 | 关闭当前激活的标签页 |
| 关闭其它 | 关闭除当前标签页外的所有标签页 |
| 关闭全部 | 关闭所有标签页 |

### 3.4 变量引用规则

- 全平台统一使用 `{{变量名称}}` 语法引用变量和参数。
- 支持动态传参与拼接，例如: `source_table_{{random(12)}}`。
- 变量来源包括: 环境变量、全局变量、提取变量、全局参数函数。

### 6.1 项目管理

#### 6.1.1 功能概述

管理项目基础信息和数据库配置信息。涉及两张核心表：**接口项目表**、**数据库配置表**（通过 `project_id` 关联）。

#### 6.1.2 项目列表

**页面类型**: 列表页（支持分页）

**列表字段**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 项目名称 | 文本 | — |
| 项目描述 | 文本 | — |
| 创建时间 | 日期时间 | — |
| 更新时间 | 日期时间 | — |
| 操作 | 操作列 | 见下方操作说明 |

**操作列**:

| 操作 | 形式 | 说明 |
|------|------|------|
| 编辑 | 图标按钮 | 打开编辑弹窗，修改项目名称、描述 |
| 数据库配置 | 图标按钮 | 下钻进入数据库配置页面 |
| 删除 | 图标按钮 | 点击后触发**二次确认弹窗**，确认后删除 |

**交互说明**:
- 增删改操作完成后显示全局 Toast 提示。

#### 6.1.3 数据库配置（下钻页面）

**入口**: 项目列表 → 点击「数据库配置」图标。

**页面类型**: 列表页（支持分页）

**列表字段**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 连接名称 | 文本 | 用户自定义的连接标识 |
| 引用变量 | 文本 | 可通过 `{{引用变量}}` 在其他模块引用该数据源 |
| 配置信息 | 文本 | 保存后自动组装展示（如 `host:port/db`） |
| 连接状态 | 状态标签 | 默认每 **10 分钟**自动测试连接一次 |
| 启用状态 | 开关按钮 | 默认**启用**；可通过开关切换启用 / 关闭 |
| 操作 | 操作列 | 编辑、删除 |

**适配数据库类型**:

| 数据库类型 | 优先级 |
|------------|--------|
| MySQL | 当前版本支持 |
| PostgreSQL | 当前版本支持 |

**新增 / 编辑弹窗 — 表单字段**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 连接名称 | 文本输入 | 是 | — |
| 引用变量 | 文本输入 | 是 | — |
| 数据库类型 | 下拉选择 | 是 | MySQL / PostgreSQL |
| 主机 | 文本输入 | 是 | — |
| 端口 | 数字输入 | 是 | — |
| 初始数据库 | 文本输入 | 是 | — |
| 用户名 | 文本输入 | 是 | — |
| 密码 | 密码输入 | 是 | — |

**业务规则**:

| 规则编号 | 规则描述 |
|----------|----------|
| DB-001 | 表单底部提供「测试连接」按钮 |
| DB-002 | **必须测试连接成功后**才能保存提交 |
| DB-003 | 保存成功后自动组装配置信息字段 |
| DB-004 | 新增记录默认启用 |
| DB-005 | 数据库配置表关联项目表（`database.project_id = project.id`） |
| DB-006 | 连接状态默认每 10 分钟自动刷新（后端定时任务） |

### 6.2 关键字配置

#### 6.2.1 功能概述

管理可扩展的关键字函数，支持用户自定义关键字并在后续场景编排等模块的下拉框中引用。编辑器使用 **Monaco 编辑器**。

#### 6.2.2 关键字列表

**页面类型**: 列表页

**列表字段**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 关键字类型 | 枚举 | 分类标识 |
| 名称 | 文本 | 关键字显示名 |
| 方法名 | 文本 | 函数方法名 |
| 启用状态 | 开关按钮 | 禁用后不在下拉框中显示 |
| 操作 | 操作列 | 编辑、删除（内置关键字无此操作） |

#### 6.2.3 新增关键字

**表单字段**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 关键字类型 | 下拉选择 | 是 | — |
| 名称 | 文本输入 | 是 | — |
| 方法名 | 文本输入 | 是 | — |
| 代码块 | Monaco 编辑器 | 是 | 关键字函数代码 |
| 入参释义 | 动态表格 | 否 | 参数名 + 描述，支持多行 |

**业务规则**:

| 规则编号 | 规则描述 |
|----------|----------|
| KW-001 | 新增关键字保存后**默认启用** |
| KW-002 | 禁用后，该关键字不再出现在其他模块的下拉选项中 |

#### 6.2.4 内置关键字

> 内置关键字**不可删除、不可编辑**，但需要将代码**回显在前端页面**供用户参考。

| 内置关键字 | 功能说明 |
|------------|----------|
| **发送请求** | 支持 requests 库所有请求方式（GET、POST、PUT、DELETE、PATCH 等） |
| **断言类型** | 断言对象支持：JSON、Header、Cookie、环境变量、HTTP Code、耗时等 |
| **提取变量** | 提取对象支持：JSON、Header、Cookie；支持 JSONPath 提取 |
| **数据库操作** | 支持选择对应项目下的数据源，输入 SQL 命令；支持数据库断言和提取变量 |
| **自定义操作** | 占位符（预留扩展） |

### 6.3 接口定义

#### 6.3.1 功能概述

以项目为维度，定义和管理 API 接口信息，并提供快速调试能力。支持环境管理和变量引用。

#### 6.3.2 接口导入方式

| 导入方式 | 说明 |
|----------|------|
| 手动填写 | 逐项填写接口信息 |
| cURL 导入 | 粘贴 cURL 命令自动解析 |
| Swagger 导入 | 通过 Swagger/OpenAPI 文档批量导入（卡片形式） |

#### 6.3.3 目录与接口管理

| 功能 | 说明 |
|------|------|
| 新建目录 | 支持创建多级目录结构 |
| 放到指定目录 | 接口可归属到目录下 |
| 拖拽排序 | 目录和接口均支持拖拽排序 |

#### 6.3.4 文件参数处理

| 规则编号 | 规则描述 |
|----------|----------|
| API-FILE-001 | 接口调试中的 `file` 类型参数，需通过 MinIO 上传 |
| API-FILE-002 | MinIO 需根据**不同项目**将文件上传到**不同的桶（Bucket）** |

#### 6.3.5 环境管理

**入口**: 接口定义页面右上角，显示当前环境名称的下拉框 + 旁边的「环境管理」按钮。

**环境管理弹窗**:

| 功能区域 | 说明 |
|----------|------|
| 全局变量 | 所有环境共享；支持手动设置、通过提取变量设置 |
| 环境变量 | 仅当前环境可用；需通过「新建环境」后设置 |

**新建环境 — 表单字段**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 环境名称 | 文本输入 | 是 | — |
| 前置 URL | 文本输入 | 是 | 如 `https://api.example.com` |
| 环境变量 | 动态表格 | 否 | 变量名 + 变量值 + 变量描述 |

**变量引用**: 在接口参数中通过 `{{变量名}}` 引用全局变量或环境变量。

### 6.4 场景编排

#### 6.4.1 功能概述

将多个接口步骤编排为一个完整的测试场景。一个场景对应一个 YAML 文件。涉及两张表：**场景表**、**步骤表**。需与项目 ID 关联。

#### 6.4.2 场景列表

**页面类型**: 列表页（支持分页）

**查询条件**:

| 查询字段 | 类型 | 说明 |
|----------|------|------|
| 测试场景名称 | 文本输入 | 模糊查询 |
| 所属项目 | 下拉选择 | 精确筛选 |

**列表字段**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 所属项目 | 文本 | — |
| 测试场景名称 | 文本 | — |
| 测试场景描述 | 文本 | — |
| 操作 | 操作列 | 编辑、删除 |

#### 6.4.3 新增 / 编辑测试场景

场景编辑分为两大区域：**基础信息**和**场景信息**。

##### 基础信息

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| 所属项目 | 下拉选择 | 是 | — |
| 场景名称 | 文本输入 | 是 | — |
| 场景描述 | 文本输入 | 否 | — |
| 优先级 | 下拉选择 | 是 | P0 / P1 / P2 / P3，默认 P2 |
| 标签 | 标签输入 | 否 | 支持多标签 |

##### 场景信息

场景信息包含以下四部分：

| 区域 | 说明 |
|------|------|
| 测试步骤 | 主要内容区，见 6.4.4 |
| 变量定义 | 场景级变量定义 |
| 前置 SQL | Monaco 编辑器；用于用例执行前的数据准备（如建表）；支持 `{{变量名}}` 引用 |
| 后置 SQL | Monaco 编辑器；用于用例执行后的数据清理（如删表）；支持 `{{变量名}}` 引用 |

**辅助功能按钮**:

| 按钮 | 说明 |
|------|------|
| 环境管理 | 快速查看已保存的全局变量、环境变量（名称、值、描述） |
| 全局参数 | 快速浏览「系统设置 → 全局参数」中配置的工具函数 |

#### 6.4.4 测试步骤

**页面类型**: 可拖拽排序的列表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 步骤描述 | 文本输入 | — |
| 关键字操作 | 级联下拉框 | 层层筛选出需要的关键字函数 |
| 关键字参数 | 动态表单 | 根据所选关键字动态加载，见下方详细说明 |
| 操作 | 操作列 | 编辑、删除 |

**关键字选择后的动态参数加载规则**:

| 关键字选择 | 动态加载的参数 |
|------------|----------------|
| 发送请求 → HTTP 请求 | 展示该项目「接口定义」中已保存的HTTP请求列表（支持「目录 → 接口」级联筛选，也支持按接口名称搜索选择） |
| 断言类型 → JSON | 加载 JSONPath 表达式输入、判断符号选择、预期结果输入 |
| 提取变量 → JSON | 加载变量名称输入、变量类型选择（全局 / 环境）、JSONPath 表达式输入 |
| 数据库操作 | 加载数据源选择（当前项目下）、SQL 输入 |
| 其他关键字 | 按对应关键字定义的入参动态加载 |

**业务规则**:

| 规则编号 | 规则描述 |
|----------|----------|
| STEP-001 | 测试步骤支持**拖拽**改变执行排序 |
| STEP-002 | 前置/后置 SQL 支持变量和参数引用（`{{变量名或参数名}}`） |
| STEP-003 | 参数名称支持动态传参和拼接，如 `source_table_{{random(12)}}`；sisyphus-api-engine 核心执行器需完善该逻辑 |

#### 6.4.5 右侧边栏

| 功能 | 说明 |
|------|------|
| **运行环境** | 下拉框切换运行环境；旁边按钮可打开「环境管理」弹窗 |
| **测试数据集** | 下拉框选择数据集（默认「不使用测试数据」）；旁边按钮点击后展开新建数据集抽屉 |

**测试数据集 — 新建数据集**:

| 功能 | 说明 |
|------|------|
| 数据集名称 | 文本输入，必填 |
| 测试数据 | 表格形式（含表头和数据行）；支持**手动输入**、**导入 CSV**、**导出 CSV** |
| 导出 CSV | 无数据时可作为**导入模板**下载 |

**数据驱动规则**:

| 规则编号 | 规则描述 |
|----------|----------|
| DDT-001 | 数据集中每一行数据等价于一个独立的 YAML 文件 |
| DDT-002 | 数据驱动执行时**并行运行**（如 5 行数据 = 5 个并行任务） |

## 8. 核心执行器: sisyphus-api-engine

> **引擎已作为独立 PyPI 包发布** (`pip install sisyphus-api-engine`)，详细文档参见 [docs/api-engine/](./api-engine/)。

### 8.1 概述

sisyphus-api-engine 是平台的**接口自动化核心执行器**，以独立 PyPI 包形式发布，以 CLI 方式驱动，通过 YAML 文件定义测试场景，支持多种报告格式输出。

### 8.2 核心设计原则

| 原则 | 说明 |
|------|------|
| 一个 YAML = 一个测试场景 | 每个 YAML 文件代表一个完整的测试场景用例 |
| CLI 驱动 | 通过命令行调用执行，如 `sisyphus-api-engine --case xxx.yaml -O json` |
| 多格式输出 | 支持文本报告（默认，使用 rich 库）、JSON 响应、Allure 报告、HTML 报告 |
| 独立发布 | 作为独立 PyPI 包发布，后端通过 `pyproject.toml` 声明依赖 |

### 8.3 数据驱动支持

| 方式 | 说明 |
|------|------|
| CSV 文件引入 | 通过指定 CSV 文件路径引入测试数据 |
| YAML 内定义 | 在 YAML 文件中使用 `ddts` 字段定义数据集 |
| 实现方式 | 底层通过 pytest 的 `@pytest.mark.parametrize` 装饰器实现 |

### 8.4 平台集成流程

```
前端输入参数 → 后端接收参数 → 后端转化为规范 YAML 文件
    → 保存至 temp 临时目录（UUID 命名）
    → 调用 sisyphus-api-engine CLI 执行 (已通过 pip 安装)
    → 获取 JSON 响应数据
    → 返回前端展示
```

**关键约定**:

| 约定项 | 说明 |
|--------|------|
| 引擎安装 | 通过 `pip install sisyphus-api-engine` 安装，已在后端 `pyproject.toml` 中声明依赖 |
| YAML 存储 | 保存在 `temp/` 临时目录中，文件名为唯一 UUID |
| 交互规范 | 后端需预先定义 YAML 输入规范和 JSON 输出规范的文档 |
| 前端展示 | 根据 JSON 响应中的丰富参数在页面中展示指标和报告 |
